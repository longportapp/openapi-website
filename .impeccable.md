## Design Context

### Users
Quantitative developers, fintech engineers, and AI agent builders who need programmatic access to real-time market data and trading across HK, US, CN, SG markets. They work in terminals and IDEs, value speed and precision, and increasingly use AI tools (Claude, Cursor) as coding assistants.

### Brand Personality
**Technical but warm** — like bun.sh or Supabase. Not cold corporate, not frivolous. The interface should make developers want to try things immediately. Three words: **precise, inviting, capable**.

### Aesthetic Direction
- **Tone**: Developer Playful — typing animations, brand color accents, code-first layouts that feel alive
- **References**: bun.sh (structure, typing animation, compact product grid), Supabase (social proof, product cards), public.com (large numbers, warm dark backgrounds)
- **Anti-references**: Bloomberg Terminal (too dense), generic SaaS landing pages (too marketing), Tiger Brokers (high-saturation clutter)
- **Theme**: Light mode primary, dark mode as equal citizen. Code blocks always use dark background for contrast
- **Brand color**: `#00B8B8` (teal/cyan) — must be used sparingly as accent, not flooding the UI

### Design Principles
1. **Code is the hero** — every section shows real, copyable, working commands
2. **One glance, four products** — OpenAPI, MCP, CLI, SKILL must be immediately visible and distinguishable
3. **Respect developer time** — no filler content, no decorative-only elements, every pixel earns its place
4. **Typing is trust** — the terminal typing animation is the signature interaction, showing real capability
5. **LBUS token compliance** — all colors via CSS variables, rounded-md (6px), shadow-card, no hardcoded values

### Technical Constraints
- VitePress 2.0 + Vue 3 + UnoCSS (presetWind3)
- LBUS design tokens from longbridge-websites (CSS variables in css-var.scss)
- Three locales: en, zh-CN, zh-HK
- No React — Vue components only
- Static assets on CDN, no images in repo
