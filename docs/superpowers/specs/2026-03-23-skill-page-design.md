# Skill 页面设计文档

**日期**：2026-03-23
**路由**：`/${lang}/skill`（例如 `/zh-CN/skill`、`/zh-HK/skill`、`/skill`（英文 root））
**技术**：Vue 组件（VitePress），与 SDK 页面模式一致（`<Skill />`）

---

## 背景与目标

### 背景

Longbridge Developers 平台目前提供 API、SDK、MCP、CLI 等多种接入方式，面向有研发能力的投资者。但平台现在需要服务一类新的用户——**普通投资者**：他们不懂编程，但在使用 OpenClaw、ChatGPT、Claude 等 AI 工具，希望通过 AI 助手直接访问金融数据与交易能力。

**SKILL** 是这类用户的入口：一个可以安装到 AI 工具中的插件，安装后 AI 助手就能调用 Longbridge 的完整平台能力（数据、API、SDK、MCP、CLI）。

### 目标

- 面向**不懂编程的普通投资者**，以用户价值而非平台能力为视角
- 清晰呈现 SKILL 能帮投资者做什么（场景驱动）
- 为各主流 AI 客户端提供差异化的安装引导（OpenClaw 优先）
- 通过动画对话演示，让用户直观感受 SKILL 的实际效果

---

## 目标用户

普通投资者，使用以下 AI 工具之一：

| 优先级 | 客户端 |
|--------|--------|
| 主推 | OpenClaw |
| 次要 | ChatGPT、Claude |
| 补充 | Claude Code、Codex |

## 页面结构

### 导航

在顶部导航栏现有项目（首页 / SDK / 文档 / MCP / CLI / LLM）后新增 **SKILL** 入口。

各 locale 的 `nav.ts` 均需添加：
```ts
{ text: 'Skill', link: `/${lang}/skill`, activeMatch: `^/${lang}/skill` }
```
文件路径：`docs/zh-CN/skill.md`、`docs/zh-HK/skill.md`、`docs/en/skill.md`。

---

### 1. Hero 区

**目标**：第一屏抓住用户注意，传达核心价值，提供安装入口。

#### 内容

- **Badge**：`Longbridge Developers · Skill`（无 emoji）
- **主标题**：让 AI 直接查行情、做交易、读财报
- **副标题**：无需编程。通过 Longbridge Skill，让你的 AI 助手（OpenClaw、ChatGPT、Claude 等）直接访问港美股实时数据与交易能力。
- **CTA 区**（纵向排列，最大宽度 480px，居中）：
  1. **ZIP 下载按钮**（主，紫色填充）+ "推荐" 标签
  2. 分隔线："或命令行安装"
  3. **npx 命令框**：全宽代码块 + 右侧复制按钮
  4. 文字链接："查看各客户端安装指引"
- **客户端标识行**：OpenClaw（金色突出）/ ChatGPT / Claude / Claude Code / Codex

#### 设计规范

- ZIP 按钮优先于 npx 命令，因为目标用户多数不熟悉命令行
- npx 命令框使用等宽字体，全宽显示，确保长命令不折行
- 无 emoji，图标使用 SVG icon 或纯文字
- OpenClaw 使用金色（`#f59e0b`）区别其他客户端

### 2. 场景介绍区

**目标**：以投资者行为视角介绍 SKILL 的五大使用场景，放置于 Hero 之后、交互演示之前，作为演示的铺垫。

#### 布局

5 列等宽卡片，横排一行。

#### 卡片内容结构

每张卡片包含：
1. **场景标签**（小号大写，带颜色区分）
2. **场景标题**（投资者视角，说明能做什么）
3. **场景说明**（2-3 句，描述具体能力）
4. **示例问法**（用引用样式展示真实提示词，帮助非程序员知道怎么用）

#### 五个场景

| # | 标签 | 标题 | 示例问法 | 颜色 |
|---|------|------|----------|------|
| 1 | 行情监控 | 实时掌握市场动态，洞察资金与盘口 | "今天科技板块资金净流入最多的是哪几只？主力大单方向怎样？" | `#6366f1` |
| 2 | 个股全景分析 | 读懂一家公司，从财报到估值到新闻 | "分析 NVDA：最新季报亮点、当前估值是否合理，以及主要风险" | `#a78bfa` |
| 3 | 智能交易 | 一句话完成下单、改单、查持仓 | "以市价买入 100 股 AAPL，执行前告诉我当前价格" | `#f59e0b` |
| 4 | 走势 & 技术分析 | 用历史数据，找到买卖的时机参考 | "TSLA 近 3 个月走势怎样？当前在关键支撑位还是压力位附近？" | `#22d3ee` |
| 5 | 持仓 & 账户 | 随时了解你的资产全貌与盈亏状况 | "我今天持仓盈亏怎样？哪只股票贡献最多，哪只在拖累组合？" | `#4ade80` |

### 3. 交互演示区

**目标**：通过动画对话模拟，让用户直观感受在真实 AI 工具中使用 SKILL 的效果。

#### 布局（方案 B：双轴切换）

```
┌─────────────────┬──────────────────────────────────────┐
│  场景列表（左）  │  客户端 Tab + 对话窗口（右）          │
│                 │                                       │
│  > 实时行情     │  [OpenClaw] [ChatGPT] [Claude] [...]  │
│    智能交易     │  ┌──────────────────────────────┐     │
│    行情订阅     │  │ 模拟客户端对话窗口             │     │
│    财报分析     │  │ 用户消息 →                    │     │
│                 │  │ ← Tool Call（可展开）          │     │
│                 │  │ ← AI 回复（逐字打出）          │     │
│                 │  └──────────────────────────────┘     │
└─────────────────┴──────────────────────────────────────┘
```

#### 交互行为

- **切换场景**：点击左侧场景列表，右侧对话内容重置并重新播放动画
- **切换客户端**：点击右上客户端 Tab，对话窗口切换为对应客户端的 UI 风格，内容重新播放
- **动画效果**：
  - 用户消息：直接出现
  - Tool Call：展开动画 → 显示命令 → 收起
  - AI 回复：打字机效果逐字输出
  - 数据行（股价、涨跌幅）：数字滚动呈现

**注意**：演示区的 4 个场景与场景介绍区的 5 张卡片是两套独立内容，各自服务不同目的——场景介绍区是静态总览卡片（覆盖更广的投资场景），演示区是有交互动画的深度展示（专注最具代入感的 4 个场景）。两者不需要一一对应。

#### 四个演示场景

| 场景 | 用户问法示例 | 演示重点 |
|------|-------------|---------|
| 实时行情 | NVDA 和 TSLA 现在行情怎么样？ | 多标的实时报价 + 涨跌展示 |
| 智能交易 | 帮我以市价买入 100 股 AAPL | 确认流程 + 执行结果 |
| 行情订阅 | 订阅 TSLA 实时行情推送 | 订阅确认 + 推送示意 |
| 财报分析 | 对比 TSLA 和 NVDA 最新季报 | 财报拉取 + 数据对比表 |

#### 客户端 UI 风格

每个客户端的对话窗口 Chrome（标题栏、按钮、输入框样式）模拟对应工具的真实界面风格：
- **OpenClaw**：默认激活，金色主题标识
- **ChatGPT / Claude**：各自的配色与界面特征
- **Claude Code / Codex**：终端/代码风格

---

### 4. 安装区

**目标**：提供简洁、按客户端差异化的安装引导，不抢演示风头。

#### 布局

- 区块标题：**开始安装**，副标题：选择你使用的 AI 工具
- 客户端横向 Tab（与演示区保持一致的顺序）
- 面板内容随 Tab 切换，**两种模板**：

**模板 A：OpenClaw**
> 在 OpenClaw 对话框中输入以下内容，即可完成安装：
> ```
> /skills add longbridge/longbridge-terminal
> ```
> （代码框 + 复制按钮）

**模板 B：其余所有客户端**（ChatGPT / Claude / Claude Code / Codex）
1. ZIP 下载按钮（主，带"推荐"标签）
2. 分隔线："或命令行安装"
3. npx 命令框（全宽，带复制按钮）

## 技术实现要点

- 页面入口：`docs/zh-CN/skill.md`（及 `en/`、`zh-HK/` 对应文件），使用 `<Skill />` 自定义 Vue 组件
- 导航：在 `zh-CN/nav.ts`、`en/nav.ts`、`zh-HK/nav.ts` 中新增 SKILL 项
- 动画对话组件：独立封装 `<SkillDemo>` 组件，接受场景 + 客户端两个 props 控制内容。**必须用 `<ClientOnly>` 包裹或在 `onMounted` 内初始化动画**，避免 SSR 水合错误（项目近期已有相关修复记录）
- 安装面板：独立封装 `<InstallPanel>` 组件，按 `client` prop 渲染不同模板
- 场景卡片：静态数据驱动，抽成配置数组

---

## 多语言

页面需提供三个语言版本：`zh-CN`、`zh-HK`、`en`。文案内容、场景示例均需本地化。动画演示中的示例问法也需对应翻译。
